# Design notes

Single-issue **implementation design gates** — design docs scoped to one issue or
PR that informed an implementation but are not enhancement proposals. They keep
descriptive slug filenames and carry **no SFEP number**.

These are kept for the record and for tracing why a specific implementation took
the shape it did. For the proposal process and what counts as an SFEP, see
[`../0001-sfep-process.md`](../0001-sfep-process.md) §1 and §9.
